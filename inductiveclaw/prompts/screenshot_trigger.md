If the project has a visual component, use take_screenshot to capture the current
state, then Read the screenshot file to inspect it visually.

When evaluating the screenshot, check:
- Layout: Is content well-spaced? Any overlapping elements or overflow?
- Typography: Readable font sizes? Consistent hierarchy? Good contrast?
- Color: Cohesive palette? No clashing colors? Dark/light mode if applicable?
- Responsiveness: Would this look broken at other screen sizes?
- Polish: Does this look intentionally designed or auto-generated?

If something looks off, fix it before moving on.
